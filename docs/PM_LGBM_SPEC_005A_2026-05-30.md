# PM-LGBM-SPEC-005A

Date: 2026-05-30
Owner: PM route control
Status: implementation-approved by user after this spec

## Goal

Unblock Ian's LightGBM route with the smallest validation-only adapter. This is
not a full rerun, not a model-quality claim, and not a paper/evidence update.

## Decision

Use the existing local runner and prepared-data path. Add `lightgbm==4.6.0` to
the pinned dependencies and implement a lazy-import LightGBM validation-only
model family beside the existing `sklearn_logreg` path.

The adapter must reuse:

- `mentor_clean_v1`
- `no_trade_band`
- `threshold_source=fixed_pre_registered_5bps`
- `decision_time_policy=post_bar_close_completed_bar`
- `scaler_id=standard_pooled_train_only_v1`
- validation-only result embargo for test metrics

## Allowed Files

- `docs/PM_LGBM_SPEC_005A_2026-05-30.md`
- `requirements.txt`
- `scripts/phase1b_local/local_baseline_matrix.py`
- `tests/test_phase1b_local_runner.py`

## Forbidden Actions

- No notebook execution.
- No torch training.
- No full-run baseline.
- No evidence matrix, wiki, Zotero, or paper artifact changes.
- No source data changes.
- No dependency installation unless separately needed for an actual runtime
  smoke. Unit tests should use a fake LightGBM module so the code path can be
  verified without network or package installation.

## Implementation Rules

- Keep the LightGBM import lazy inside a helper so normal imports and tests do
  not fail when the package is absent.
- Do not rename `sklearn_logreg` or use it as LightGBM.
- Use validation macro F1 for validation-only readout only; do not read or
  report test metrics in validation-only mode.
- Use deterministic, small default LightGBM parameters:
  `objective=binary`, `n_estimators=100`, `learning_rate=0.05`,
  `num_leaves=31`, `random_state=42`, `n_jobs=1`, `verbosity=-1`.
- Result rows must identify `model_name=lightgbm_lgbm_classifier` and
  `model_family=lightgbm`.
- If the real package is missing, fail with an explicit install/dependency
  message rather than silently falling back.

## Test Gates

- `py_compile` for the runner and test file.
- Targeted runner pytest must pass.
- Existing scaler/no-trade/window guardrail pytest should still pass.
- No real LightGBM training smoke is required until the dependency is installed.

## Stop Rules

- Stop if implementation needs changes outside the allowed files.
- Stop if any path would inspect test metrics during validation-only reporting.
- Stop if dependency install is required before the user approves installation.
- Stop if a result could be confused with LightGBM full-run performance.
